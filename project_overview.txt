Shoe Manufacturing Project Overview

This is a comprehensive breakdown of the Shoe Manufacturing project, a robust, full-stack MERN (MongoDB, Express, React, Node.js) enterprise application designed to manage both the e-commerce storefront for customers and the complex, end-to-end internal operations of a shoe factory (ERP - Enterprise Resource Planning).

---
1. High-Level Architecture
---
* Frontend: Built with React 19, React Router, Vite, and Bootstrap/React-Bootstrap. Uses Context API for state management.
* Backend: Node.js and Express.js handling API routing.
* Database: MongoDB (via Mongoose ODMs) storing all business data from user credentials to material inventory.
* Authentication: JWT (JSON Web Tokens) combined with bcrypt for secure, role-based login access (user, admin, artisan).

---
2. Detailed Breakdown of Project Modules
---

A. The Customer Storefront (E-Commerce & Customization)
Located in: src/pages and src/pages/user
This is the public-facing side of the application where retail users browse, design, and purchase shoes.
* Catalog & Product Discovery (Catalog.jsx, ProductDetail.jsx): Standard e-commerce browsing of the pre-made shoe catalog.
* Bespoke Designer (CustomDesigner.jsx): A flagship feature allowing users to customize their shoes (colors, materials, sizes) before ordering.
* Shopping & Checkout (Cart.jsx, Checkout.jsx): Standard shopping cart and checkout pipeline.
* User Dashboard (UserDashboard.jsx, OrderHistory.jsx, UserOrderTracking.jsx): A dedicated customer portal to view past orders, track current order manufacturing status in real-time, and manage saved bespoke designs.

B. The Admin Portal (Enterprise Management / ERP)
Located in: src/pages/admin & supported by various backend controllers
This massive module acts as the brain of the manufacturing business, providing factory managers and administrators complete control over operations.
* Order Management (AdminOrders.jsx, AdminOrderDetail.jsx): Processing incoming customer purchases and pushing them to the production floor.
* Production & Manufacturing (AdminProduction.jsx, AdminLiveStudio.jsx): Managing production batches, assigning tasks to artisans, and tracking manufacturing stages (e.g., Cutting, Stitching, Assembly, Finishing).
* Inventory & Materials (AdminInventory.jsx, AdminSuppliers.jsx): Tracking raw materials (leather, rubber, thread) and finished goods. Handling material consumption and placing purchase orders with external suppliers.
* HR & Workforce (AdminArtisans.jsx, AdminHR.jsx, AdminAssignArtisan.jsx): Managing artisan profiles, tracking their shift clocks, calculating salaries, and assigning them to specific work orders.
* Finance & Analytics (AdminFinance.jsx, AdminAnalytics.jsx): Tracking revenue from orders versus expenditures (material purchases, salaries) to provide business intelligence.
* Quality Assurance (AdminQuality.jsx): Overseeing Quality Control (QC) reports submitted by artisans before products ship.

C. The Artisan Portal (Factory Floor Operations)
Located in: src/pages/artisan
This is a specialized interface used by the actual workers (cobblers, designers, assemblers) on the factory floor.
* Task & Production Management (ArtisanDashboard.jsx, ArtisanAssignedOrders.jsx, ArtisanProductionUpdate.jsx): Artisans see exactly what shoes they need to build today. They can update the status of a shoe.
* Time & Shift Tracking (ArtisanShiftClock.jsx): For punching in and out of shifts.
* Quality & Maintenance (ArtisanQualityCheck.jsx, ArtisanMaintenance.jsx): Submitting QC reports for completed shoes and logging maintenance issues.
* Material Requisition (ArtisanMaterials.jsx): Requesting or logging the usage of raw materials on the floor.
* Communication (ArtisanMessages.jsx, ArtisanWorkshopClips.jsx): Internal chat system and a space to upload "Clips".

---
3. Backend Data Models (backend/models/)
---
The database schema perfectly mirrors the complexity of a real factory:
* Users & Auth: User.js (Customers, Admins, Artisans)
* Sales: Product.js, Order.js, BespokeDesign.js
* Supply Chain: RawMaterial.js, Supplier.js, MaterialPurchase.js, MaterialConsumptionLog.js
* Manufacturing pipeline: ProductionBatch.js, WorkOrder.js, ProductionStage.js, StageLog.js
* Operations: QualityControlReport.js, MaintenanceLog.js, Salary.js, ShiftManagement.js

---
4. Summary of the Data Flow
---
1. A Customer uses the CustomDesigner to order a bespoke shoe.
2. The order hits the orderController and is saved.
3. An Admin reviews the order in AdminOrders and converts it into a ProductionBatch via AdminProduction.
4. The Admin uses AdminAssignArtisan to assign the batch to an Artisan.
5. The Artisan logs into their tablet on the floor, sees the task in ArtisanAssignedOrders, and begins work.
6. As the Artisan progresses, they update the ProductionStage. This logs material consumption and updates the Customer's UserOrderTracking UI.
7. Upon completion, the Artisan submits a QualityControlReport. Once passed, it is shipped!

5. Frontend Components and Architecture Insight (src/)
---

A. Components Directory (src/components/)
1. Layout Components (src/components/layout/)
Because this app has three completely different user experiences, it uses specialized layout wrapper components.
* AdminLayout.jsx: Wraps the Admin portal. Provides a heavy-duty sidebar navigation and a top header. Enforces that only users with the admin role can see the content inside.
* ArtisanLayout.jsx: Wraps the factory floor portal. Features a simplified, touch-friendly navigation suitable for a tablet used by a worker.
* UserLayout.jsx: Wraps the logged-in customer's dashboard.
* Layout.jsx & Navbar.jsx & Footer.jsx: The standard wrapper for public pages.

2. Landing Page Components (src/components/landing/)
The marketing landing page is broken down into clean, maintainable sections:
* Hero.jsx: The big, eye-catching top section.
* Process.jsx: Explains how the bespoke shoemaking process works.
* Collections.jsx: Highlights featured shoe collections.
* Testimonials.jsx: Customer reviews.
* Craftsmanship.jsx: Details about the quality and materials used.

3. UI Elements (src/components/ui/)
Reusable UI elements used everywhere to maintain a consistent design system.
* StatusBadge.jsx: A crucial component that takes a status string (like "Pending", "Cutting") and renders a nicely colored, pill-shaped badge.
* Button.jsx: A standardized button component.

B. Global State Management (src/context/)
The application uses React's Context API to manage state globally:
* AuthContext.jsx: Handles the login state. It stores the JWT token, verifies the user's role (user, admin, artisan), and exposes login() and logout() functions.
* CartContext.jsx: Manages the e-commerce shopping cart. Updates when a user clicks "Add to Cart" and is read by the Navbar and Cart page.

C. Static Data (src/data/)
* mockData.js: A very large file containing extensive placeholder data (Products, Orders, Artisans, Materials). Used extensively during early development and testing, or to seed the database.
